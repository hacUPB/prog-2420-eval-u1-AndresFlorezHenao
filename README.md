1. Función principal de cada componente investigado

•	Hardware

        - CPU (Unidad Central de Procesamiento): Es el componente principal de una computadora que realiza operaciones aritméticas, lógicas y de control para ejecutar instrucciones de los programas. Es conocido como el "cerebro" del ordenador.

        - RAM (Memoria de Acceso Aleatorio): Es una forma de almacenamiento de datos temporales que permite a la CPU acceder rápidamente a los datos e instrucciones que necesita para ejecutar programas. Es volátil, lo que significa que los datos se borran al apagar el equipo.

    - Almacenamiento (HDD, SSD):

        - HDD (Disco Duro): Dispositivo de almacenamiento magnético que utiliza platos giratorios y cabezales de lectura/escritura para almacenar datos de forma permanente.
        - SSD (Unidad de Estado Sólido): Dispositivo de almacenamiento más rápido y confiable que el HDD, utiliza memoria flash para almacenar datos de manera permanente sin partes móviles.
    
        -Placa base (Motherboard): La placa principal del ordenador que conecta todos los componentes, incluidos la CPU, RAM, almacenamiento y dispositivos periféricos. Proporciona las conexiones y el sistema de comunicación entre los componentes.

    - Dispositivos de entrada y salida:

        -Dispositivos de Entrada: Permiten al usuario interactuar con la computadora proporcionando datos e instrucciones. Ejemplos incluyen teclado, ratón, escáner.
        -Dispositivos de Salida: Permiten que la computadora comunique información al usuario. Ejemplos incluyen monitor, impresora, altavoces.
    -Buses (de datos y dirección):

        -Bus de Datos: Canal de comunicación que transfiere datos entre la CPU y otros componentes de la computadora.
        -Bus de Dirección: Canal de comunicación que transfiere las direcciones de memoria, indicando a la CPU dónde encontrar o almacenar datos.
- Software

        -Sistema Operativo: Programa fundamental que gestiona el hardware y el software de la computadora, proporcionando una interfaz entre el usuario y los componentes del sistema. Ejemplos incluyen Windows, macOS, Linux.

        -Aplicaciones de Usuario: Programas diseñados para realizar tareas específicas que beneficien al usuario. Ejemplos incluyen procesadores de texto, navegadores web, juegos.

        -Drivers: Programas que permiten al sistema operativo y a las aplicaciones comunicarse y controlar dispositivos de hardware específicos. Ejemplos incluyen controladores de impresoras, tarjetas gráficas, dispositivos USB.

        -Software de Desarrollo: Conjunto de herramientas y programas que permiten a los desarrolladores crear, depurar y mantener otros programas y aplicaciones. Ejemplos incluyen entornos de desarrollo integrados (IDEs), compiladores, depuradores.

2. Interacción del hardware con el software y función de cada uno

•	Hardware: Proporciona la infraestructura física necesaria para ejecutar el software. Incluye componentes como la CPU, memoria, almacenamiento, y dispositivos de entrada/salida.

•	Software: Consiste en programas y sistemas operativos que dan instrucciones al hardware para realizar tareas específicas.

o	Sistema Operativo (OS): Actúa como intermediario entre el usuario y el hardware, gestionando recursos y permitiendo la ejecución de aplicaciones.

o	Drivers: Programas específicos que permiten que el sistema operativo se comunique con el hardware.

o	Aplicaciones: Programas diseñados para realizar tareas específicas para el usuario, como procesadores de texto, navegadores web, etc.

3. Función de la CPU y sus partes más importantes

•	Función de la CPU: Ejecuta instrucciones de los programas mediante operaciones aritméticas, lógicas, de control y de entrada/salida. Es el componente principal para el procesamiento de datos.
•	Partes más importantes de la CPU:
o	Unidad de Control (Control Unit): Dirige el funcionamiento de la CPU y coordina la actividad de los demás componentes.
o	Unidad Aritmético-Lógica (ALU): Realiza operaciones aritméticas y lógicas.
o	Registros: Pequeñas áreas de almacenamiento dentro de la CPU que guardan datos temporales y instrucciones.
o	Cache: Memoria de alta velocidad que almacena datos e instrucciones frecuentemente usados para acelerar el acceso.

4. ¿Qué es la velocidad de la CPU, también conocida como velocidad del reloj?

Velocidad de la CPU, también conocida como velocidad del reloj, se refiere a la cantidad de ciclos de procesamiento que una CPU puede ejecutar por segundo. Se mide en Hertzios (Hz). Generalmente, se expresa en gigahercios (GHz), donde 1 GHz equivale a mil millones de ciclos por segundo.

Importancia de la Velocidad del Reloj
Rendimiento de la CPU: Una mayor velocidad de reloj permite que la CPU ejecute más instrucciones por segundo, lo que puede traducirse en un mejor rendimiento en tareas que dependen de la velocidad de procesamiento.
Comparación de Procesadores: Es una medida clave para comparar el rendimiento entre diferentes CPUs, aunque no es el único factor que influye en el rendimiento total, ya que la arquitectura, la cantidad de núcleos y la eficiencia del procesador también juegan roles importantes.
Funcionamiento
Ciclos de Reloj: Cada ciclo de reloj es una señal de pulso enviada por un oscilador interno que sincroniza las operaciones del procesador. Durante cada ciclo, la CPU puede realizar operaciones como leer instrucciones de la memoria, ejecutar cálculos y almacenar resultados.
Frecuencia del Reloj: La frecuencia del reloj determina cuántos de estos ciclos ocurren por segundo. Por ejemplo, una CPU con una velocidad de 3.5 GHz puede realizar 3.5 mil millones de ciclos de reloj por segundo.

5. ¿Cuál es la secuencia de pasos que ocurre, desde el momento en que presionas el botón de encendido de la computadora, hasta que se muestra el sistema operativo listo para funcionar? Describe todos los elementos involucrados y el paso a paso.

- Presionar el botón de encendido:

Fuente de Alimentación (PSU): Se activa y suministra energía a todos los componentes del ordenador.

- Inicialización de BIOS/UEFI:

    - BIOS (Sistema Básico de Entrada/Salida) o UEFI (Interfaz de Firmware Extensible Unificada) se inicia y realiza el POST (Power-On Self Test) para verificar el correcto funcionamiento del hardware, como la RAM, CPU, y dispositivos de almacenamiento.
    - BIOS/UEFI detecta y configura los dispositivos de hardware conectados.
- Identificación del dispositivo de arranque:

    - BIOS/UEFI busca un dispositivo de arranque (por ejemplo, HDD, SSD, o una unidad USB) donde se encuentre el sistema operativo.
    - Carga el Bootloader desde el dispositivo de arranque seleccionado.
- Carga del Bootloader:

    - El Bootloader es un programa pequeño que se carga en la RAM. Su función principal es cargar el kernel del sistema operativo en la memoria.
            - Ejemplos de bootloaders incluyen GRUB (para sistemas Linux) y el gestor de arranque de Windows (Windows Boot Manager).
- Inicialización del Kernel del Sistema Operativo:

    - El Bootloader carga el kernel del sistema operativo en la RAM y le transfiere el control.
    - El kernel inicializa los controladores de hardware necesarios para el funcionamiento del sistema, como controladores de almacenamiento, gráficos y de red.
- Configuración de los dispositivos de hardware:

    - El kernel configura y gestiona todos los dispositivos de hardware conectados, asegurando que estén listos para su uso.
    Se montan los sistemas de archivos, permitiendo el acceso a los datos almacenados en los dispositivos de almacenamiento.
- Carga de servicios y procesos del sistema:

    - El sistema operativo inicia los servicios de sistema y demonios (procesos en segundo plano) necesarios para su funcionamiento.
    Esto incluye servicios de red, administración de usuarios, gestión de seguridad, y otros procesos esenciales.
- Carga de la Interfaz de Usuario (UI):

Si el sistema operativo usa una interfaz gráfica de usuario (GUI), se inicia el entorno gráfico.
En sistemas Windows, esto incluye cargar el explorador de Windows. En sistemas Linux, puede ser el entorno de escritorio como GNOME, KDE, etc.
Si es un sistema basado en línea de comandos (CLI), se muestra la terminal o consola de comandos.
Presentación de la pantalla de inicio de sesión:

Si está configurado, el sistema operativo presenta la pantalla de inicio de sesión donde el usuario debe ingresar sus credenciales.
Una vez autenticado, el sistema carga la configuración y preferencias del usuario.
Escritorio del Sistema Operativo listo para funcionar:

El escritorio o entorno de usuario está completamente cargado y listo para la interacción.
Se pueden abrir y ejecutar aplicaciones, acceder a archivos y realizar tareas.

- Elementos involucrados:

Fuente de Alimentación (PSU): Suministra energía a los componentes.

BIOS/UEFI: Realiza POST, detecta hardware y carga el Bootloader.
Bootloader: Carga el kernel del sistema operativo.
Kernel del Sistema Operativo: Inicializa hardware y servicios básicos.

Controladores de Hardware: Permiten la comunicación entre el sistema operativo y el hardware.

Servicios y Demonios del Sistema: Procesos de fondo esenciales para el funcionamiento del sistema.

Interfaz de Usuario (GUI/CLI): Proporciona el entorno para la interacción del usuario.

Pantalla de Inicio de Sesión: Gestiona la autenticación del usuario.

Escritorio del Sistema Operativo: Entorno listo para el uso del usuario.
